# Karakeep

Karakeep is a self-hosted bookmarking and read-later service with AI-powered tagging, full-text search, and web archiving.

## Role in the infrastructure

- **Knowledge base** — Save URLs, notes, and documents with automatic content extraction, screenshot capture, and AI-generated tags.
- **VPS** — Served at `https://karakeep.<domain>` via Caddy with Cloudflare TLS.
- **Docker Desktop** — Served locally at `https://localhost:3002` via Caddy's internal TLS.

## Services

Karakeep runs as three containers that work together:

| Container | Image | Purpose |
|-----------|-------|---------|
| `karakeep` | `ghcr.io/karakeep-app/karakeep:release` | Main web application and API |
| `karakeep-chrome` | `gcr.io/zenika-hub/alpine-chrome:123` | Headless Chromium for web page screenshots and archiving |
| `karakeep-meilisearch` | `getmeili/meilisearch:v1.11.3` | Full-text search index |

`karakeep-web` depends on both `karakeep-chrome` and `karakeep-meilisearch` being available before it starts.

## Configuration

Environment is loaded from `~/.vps-launchpad/envs/karakeep.env`, which contains:

| Variable | Description |
|----------|-------------|
| `MEILI_MASTER_KEY` | Meilisearch admin key. Auto-generated at deploy time. |
| `MEILI_ADDR` | Meilisearch address (`http://karakeep-meilisearch:7700`) |
| `NEXTAUTH_SECRET` | Session signing secret. Auto-generated at deploy time. |
| `NEXTAUTH_URL` | Public URL used by NextAuth for OAuth callbacks. |
| `DATABASE_URL` | PostgreSQL connection string for the karakeep database. |
| `BROWSER_WEB_URL` | Chrome remote debugging URL (`http://karakeep-chrome:9222`) |

All secrets are auto-generated by `generate-configs.js` on first deploy and persisted in `setup-state.json`.
